Projet 4A02 : Developpement d'une application complexe

Pour ce projet nous devions créer un jeu de type ‘shoot-em-up’(lien) en 2D, nous avons donc décidé avec notre équipe de mélanger deux univers que nous apprécions, Mario Kart et Breaking Bad donnant ainsi “Breaking Kart”. Cependant contrairement aux précédents projets que nous avons pu avoir, celui-ci demandait en plus un mode de jeu en multijoueurs, cela nous a forcé à adapter notre code pour le rendre compatible avec une architecture client-serveur pour Node.Js Ce projet nous a aussi permis d'appliquer et d'approfondir nos connaissances en gestion de projet, via l'utilisation d'outils Git (Issue, Milestone, Branch). De plus, nous avons pu mettre en application sur une partie de notre projet la programmation en TDD nous permettant ainsi de couvrir plus de code et de permettre la non-régression de ce dernier lors de nos nombreuses modifications.

Compétences :

  • Node.Js et Socket.io
  • Javascript
  • TDD
  • Git
  • HTML & CSS
  • Travail en équipe (Avec Issue Git)
  • Répartition des tâches (Avec Issue Git et Milestone)
  • Estimation du temps de travail (Avec Issue Git)

Projet 4A01 : Developpement d'une API REST

Le but de ce projet etait de réaliser une API REST en Servlet en Java pour une entreprise fictive ("PizzaLand").
Cela nous a permis d'apprendre à développer une API de manière RESTful. Cette derniere permettait a ses utilisateurs de récupérer, d'ajouter, de modifier et de supprimer des ingrédients, des pizzas (contenant des ingrédients) et des commandes (contenant des pizzas) d'une base de données.
De plus ce projet nous a permis d'apprendre a utiliser et a implementer l'utilisation de "token".
Afin de visualiser nos interactions nous avons eu l'occasion d'utiliser le logiciel "Bruno" pour executer des requetes.

Compétences :

  • JavaEE (Servlet)
  • API REST
  • Utilisation d'un serveur TOMCAT
  • SQL-Requêtes

Projet 301-302 : Chasse au monstre

Pour ce projet nous devions créer un jeu de chasse au monstre en java en prenant en compte les différentes étapes de conception, telle que de l’analyse et le développement. Ce projet mêlait différents aspects de l'informatique et notamment des notions de clean code via l’intégration de différents patrons de conception (MVC et Observateur-observé) mais aussi des compétences de développement Efficace en apportant plus de réflexion sur les structures de données utilisées. Ce projet nécessitait une interface graphique en JavaFx permettant aux deux joueurs de jouer. De plus nous avons dû intégrer deux “IA” afin de pouvoir jouer seul, ces “IA” devant s’adapter à une certaine architecture afin de pouvoir s’intégrer sur les jeux des autres groupes. Nous avons décidé lors de la conception du jeu de rajouter des options visuelles et audios dérivées de licences de films ou de jeux existantes permettant plus d’immersion et de faire plaisir aux utilisateurs afin qu’ils puissent trouver le visuel qui leur correspond le plus.

Compétences :

  • Java
  • Analyse et conception de diagrammes UMLs
  • Developpement efficace
  • Clean Code
  • méthodes de calcul de plus court chemin
  • JavaFx
